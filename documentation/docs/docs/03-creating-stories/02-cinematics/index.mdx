# Кинематика

:::info[Before starting]
Это руководство предполагает, что у вас уже установлен [Basic Extension](../../02-getting-started/01-installation.mdx#basic-extension).\
Также убедитесь, что вы прочитали документацию по [layout](../../02-getting-started/02-layout.mdx) и создали страницу **cinematic**.
:::

В этом уроке вы научитесь создавать и просматривать кинематики в своих интеракциях!

## Что такое кинематики?
Кинематики — это последовательности действий по таймеру, которые можно воспроизводить в ваших интеракциях. Их используют для создания кат‑сцен и других событий с таймингом.

## Создание кинематики
Начнём с простого варианта — камеры.

### Добавление камеры
Нажмите значок + в правом верхнем углу панели и найдите `Add Camera Cinematic`. Добавьте его на страницу, кликнув по значку +.

<EntrySearch entryName='cinematic'/>

### Создание сегмента
Теперь нам нужно добавить сегмент в дорожку. Для этого выберите `New Camera Cinematic` на странице и в операциях нажмите `Add Segment`(1).

<Image img={require("../../assets/cinematics/segments.png")} alt="Adding a camera cinematic segment" width={400} />

После создания сегмента нажмите на него (2), чтобы открыть инспектор.

### Изменение сегмента камеры
В сегменте камеры есть путь. Он состоит из списка точек в мире сервера. Если указано несколько точек, игрок будет двигаться между ними, иначе останется на месте.
Чтобы создать путь, нажмите значок + в инспекторе у `path(0)`.

Чтобы задать точку пути, есть два варианта: использовать режим контента или настроить вручную.
#### Использование режима контента
В режиме контента текущие координаты игрока автоматически подставляются в поля.
<Player url={require("../../assets/cinematics/content-mode.webm").default} />

#### Ручная настройка
Можно задать точки вручную, открыв поле `Path #1`. Здесь настраиваются:
* **World** — мир, где происходит кинематика;
* **X** — координата X;
* **Y** — координата Y;
* **Z** — координата Z;
* **Pitch** — наклон камеры;
* **Yaw** — поворот камеры.

**Duration** — продолжительность от текущей точки до следующей в тиках. Если её не указать, длительность рассчитывается из оставшихся кадров.\
Например, если сегмент длится *300* кадров и содержит 3 точки, а для второй задана **duration** *200* кадров, то от первой до второй будет *100* тиков, а от второй до третьей — *200*.

Повторите шаги, чтобы добавить больше точек пути.
