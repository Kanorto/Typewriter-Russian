# Синематики

:::info[Перед началом]
Этот раздел предполагает, что у вас уже установлен [Basic Extension](../../02-getting-started/01-installation.mdx#basic-extension).\
Также убедитесь, что вы ознакомились с документацией по [расположению](../../02-getting-started/02-layout.mdx) и создали страницу **cinematic**.
:::

В этом руководстве вы научитесь создавать и просматривать синематики в своих интеракциях!

## Что такое синематики?
Синематики — это последовательности действий с привязкой ко времени, которые можно воспроизводить в интеракциях. Их используют для создания кат‑сцен или других событий по таймеру.

## Создание синематика
Начнём с простого синематика с камерой.

### Добавление синематика с камерой
Нажмите на значок + в правом верхнем углу панели и найдите `Add Camera Cinematic`. Добавьте его на страницу кликом по +.

<EntrySearch entryName='cinematic'/>

### Создание сегмента
Теперь нужно добавить сегмент в дорожку. Для этого нажмите на запись `New Camera Cinematic` на странице и в разделе операций выберите `Add Segment`(1).

<Image img={require("../../assets/cinematics/segments.png")} alt="Adding a camera cinematic segment" width={400} />

После создания сегмента нажмите на него (2) — откроется инспектор сегмента.

### Настройка сегмента камеры
Сегмент камеры содержит путь. Каждый путь — это список точек в мире сервера. Если точек несколько, игрок будет перемещаться между ними, если одна — останется на месте.
Чтобы создать путь, нажмите на значок + в инспекторе рядом с `path(0)`.

Чтобы задать точку пути, есть два варианта: воспользоваться режимом контента или настроить всё вручную.
#### Использование режима контента
Режим контента позволяет взять текущее положение игрока и заполнить поля автоматически.
<Player url={require("../../assets/cinematics/content-mode.webm").default} />

#### Ручная настройка
Вы можете настроить путь вручную, раскрыв поле `Path #1`. В нём указываются:
* **World** — мир, где происходит синематик
* **X** — координата X
* **Y** — координата Y
* **Z** — координата Z
* **Pitch** — угол наклона камеры
* **Yaw** — угол поворота камеры

**Duration** — время перехода от текущей точки к следующей в тиках. Если его не указать, длительность будет рассчитана из оставшегося времени.\
Например, если сегмент длится *300* кадров и содержит 3 точки, а у второй точки указана **duration** *200* кадров, то переход от первой к второй займёт *100* тиков, а от второй к третьей — *200* тиков.

Повторяйте процесс для добавления новых точек пути.
