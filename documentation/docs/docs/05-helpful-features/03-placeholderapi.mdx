# PlaceHolderAPI

:::warning[plugin needed]
Чтобы следовать этому руководству, необходимо установить [PlaceHolderAPI](https://www.spigotmc.org/resources/placeholderapi.6245/).
:::

## Использование плейсхолдеров PlaceHolderAPI

Если вы хотите расширить историю плейсхолдерами PlaceHolderAPI, убедитесь, что значок PlaceHolderAPI расположен над текстовым полем (см. изображение):

<Image img={require("../assets/placeholderapi/papi-icon.png")} alt="PlaceholderAPI Icon" width={400} />

Используя формат PlaceHolderAPI, вы можете задействовать их. Например, `%player_name%` выдаст имя игрока.

:::danger[Not working]
Если плейсхолдеры не работают, убедитесь, что нужный плейсхолдер установлен. Например, для `%player_name%` необходимо установить `player` через ecloud PlaceHolderAPI командой `/papi ecloud download player`.
:::

## Использование плейсхолдеров Typewriter

Typewriter предоставляет два плейсхолдера для PlaceHolderAPI:

- `%typewriter_entryid%`
- `%typewriter_entryname%`

Оба плейсхолдера дают одинаковую информацию, однако получить её можно по‑разному. Обычно рекомендуется использовать ID записи для более надёжного результата, хотя можно использовать и имя.

Чтобы найти ID записи, обратитесь к следующему изображению:

<Image img={require("../assets/placeholderapi/papi-factid.png")} alt="PlaceholderAPI Icon" width={500} />

Ваш плейсхолдер для Typewriter будет выглядеть примерно так: `%typewriter_W2X2ZbG0pzXGsS6%`.\
При вызове плейсхолдера происходит проверка ID, и:

- Если запись — персонаж, возвращается его отображаемое имя.
- Если запись — факт, возвращается значение факта.
- Если запись — звук, возвращается ID звука из ресурс-пака.
- Если запись — сущность, возвращается отображаемое имя сущности.
- Если запись — набор строк, возвращается содержимое строк.
- Если запись — квест, возвращается его отображаемое имя.
- Если запись — цель, возвращается отформатированное имя цели.
- Если запись — боковая панель, возвращается её заголовок.

### Параметры плейсхолдера
Иногда требуется получить различную информацию из записи. Это возможно с помощью параметров плейсхолдера. Ниже приведён неполный список параметров:

#### Фактовые записи
- `%typewriter_<id записи>:remaining:<число>%`: Возвращает `<число> - значение факта` (удобно, когда нужно показать, сколько ещё осталось собрать).
- `%typewriter_<id записи>:time:lastUpdated%`: Показывает время обновления факта.
- `%typewriter_<id записи>:time:lastUpdated:relative%`: Показывает прошедшее время с момента обновления факта, например `3m 20s`.

#### Временны́е факты

- `%typewriter_<id записи>:time:expires%`: Время, когда факт истекает.
- `%typewriter_<id записи>:time:expires:relative%`: Показывает оставшееся время до истечения факта, например `3m 20s`.

## Пользовательские плейсхолдеры
В Typewriter также есть два собственных плейсхолдера:

- `%typewriter_tracked_quest%`: Показывает отображаемое имя отслеживаемого квеста.
- `%typewriter_tracked_objectives%`: Показывает названия активных целей отслеживаемого квеста.
