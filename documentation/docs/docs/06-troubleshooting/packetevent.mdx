# Устранение неполадок PacketEvents

Если при включении Typewriter появляется ошибка
`java.lang.NoClassDefFoundError` с упоминанием `io.github.retrooper.packetevents`,
вероятно используется несовместимая версия PacketEvents.

## Сообщение об ошибке

Пример возможного сообщения:

```bash
[12:41:54 ERROR]: Error occurred while enabling Typewriter v0.5.1 (Is it up to date?)
java.lang.NoClassDefFoundError: io/github/retrooper/packetevents/bstats/Metrics$CustomChart
        at me.gabber235.typewriter.entry.entity.EntityHandler.initialize(EntityHandler.kt:23) ~[typewriter (5).jar:?]
        at me.gabber235.typewriter.Typewriter.onEnableAsync(Typewriter.kt:121) ~[typewriter (5).jar:?]
```
## Причина ошибки

Проблема возникает, когда установленная версия PacketEvents не подходит для вашей версии Typewriter.
**Typewriter v0.5.1 требует PacketEvents 2.5.x.** Новые версии могут быть несовместимы из‑за изменений API.

### Как исправить

1. **Удалите** текущий PacketEvents из папки `plugins` вашего сервера.
2. **Скачайте** PacketEvents версии **2.5.0**
   [по этой ссылке](https://modrinth.com/plugin/packetevents/version/QLgJReg5).
3. **Поместите** скачанный файл в папку `plugins`.
4. **Перезапустите** сервер, чтобы изменения вступили в силу.

## Проверка результата

После выполнения этих шагов Typewriter должен загружаться без ошибок.
Проверьте журналы сервера и убедитесь, что сообщений `NoClassDefFoundError` больше нет.
