# Устранение неполадок PacketEvents

Если при запуске Typewriter возникают ошибки вида `java.lang.NoClassDefFoundError`, связанные с `io.github.retrooper.packetevents`, вероятно, установлена несовместимая версия плагина PacketEvents.

## Сообщение об ошибке

Пример возможного сообщения об ошибке:

```bash
[12:41:54 ERROR]: Error occurred while enabling Typewriter v0.5.1 (Is it up to date?)
java.lang.NoClassDefFoundError: io/github/retrooper/packetevents/bstats/Metrics$CustomChart
        at me.gabber235.typewriter.entry.entity.EntityHandler.initialize(EntityHandler.kt:23) ~[typewriter (5).jar:?]
        at me.gabber235.typewriter.Typewriter.onEnableAsync(Typewriter.kt:121) ~[typewriter (5).jar:?]
```
## Причина проблемы

Эта ошибка возникает из-за того, что установленная версия PacketEvents не совместима с используемой версией Typewriter. **Typewriter v0.5.1 требует PacketEvents версии 2.5.x.** Использование более новой версии PacketEvents может привести к несовместимости из‑за изменений API.

### Как исправить

1. **Удалите** текущую версию PacketEvents из папки `plugins` вашего сервера.

2. **Скачайте** PacketEvents версии **2.5.0** [по этой ссылке](https://modrinth.com/plugin/packetevents/version/QLgJReg5).

3. **Поместите** скачанный jar‑файл PacketEvents в папку `plugins` на сервере.

4. **Перезапустите** сервер, чтобы применить изменения.

## Проверка

После выполнения этих шагов Typewriter должен загружаться без ошибок. Проверьте логи сервера, чтобы убедиться, что сообщения `NoClassDefFoundError` больше не появляются.
