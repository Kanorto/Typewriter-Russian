# Руководство по устранению проблем с загрузкой панели

Панель плагина Typewriter не загружается? Это руководство поможет найти причину и правильно настроить плагин для корректной работы.

## Правильное назначение портов

Если панель не загружается, возможно, не назначены нужные порты. Плагину Typewriter требуется два открытых порта:

1. Один порт — для загрузки панели.
2. Второй порт — для работы веб-сокета, который связывается с сервером.

:::warning
Хостинг‑площадки вроде Aternos, предоставляющие только один порт, несовместимы с Typewriter из-за этого требования.
:::

## Использование правильного URL

Убедитесь, что вы используете правильный адрес для доступа к панели. Формат должен быть `ip:port`, либо, если используется поддомен, `url:port`. Правильный URL необходим для успешного подключения.

## Как избежать дублирования портов в конфигурации

Распространённая ошибка — указание порта в поле `hostname` файла `config.yml`.
В `hostname` должно быть только IP‑адрес или доменное имя, без порта.
Пример:

```yaml title="config.yml"
# ...
// highlight-red
hostname: "127.0.0.1:25565" // Incorrect line
// highlight-green
hostname: "127.0.0.1"       // Correct line
# ...
```

Удаление порта из поля `hostname` может решить проблемы с загрузкой панели.
